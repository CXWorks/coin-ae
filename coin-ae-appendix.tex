\section{Artifact Appendix}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\subsection{Abstract}

This artifact reproduces the main results of \textsc{Coin}, a Modular
Unsafe Function (MUF) detector and PoC generator for Rust:
(C1)~the fine-tuned Llama~3.2-3B classifier achieves AUPRC\,$\approx$\,0.82
on MUF detection;
(C2)~the fine-tuned PoC generator produces valid PoCs for Modular Unsafe
Functions detected in real-world crates;
and (C3)~\textsc{Coin} substantially outperforms both vanilla open-source
LLMs (Llama 3.2 3B, Llama 3.1 8B, Qwen3 4B) and closed-source GPT-4o /
Claude-3.7 with few-shot prompting.
The artifact ships:
(i)~the Llama~3.2-3B LoRA \emph{classifier adapter} ($\approx$93\,MB),
(ii)~the Llama~3.2-3B LoRA \emph{PoC generator adapter},
(iii)~self-contained evaluation scripts in \texttt{code/} including
baseline drivers for open-source models and OpenAI/Anthropic APIs,
(iv)~five convenience launchers in \texttt{scripts/} (env setup,
smoke test, full eval, PoC eval, baseline comparison), and
(v)~a reassemblable sample test split and a starter few-shot file.
Both adapters load on top of \texttt{unsloth/Llama-3.2-3B-bnb-4bit} from
HuggingFace.
The code is released under the MIT license; model weights follow the
Llama~3.2 Community License.

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\subsection{Description \& Requirements}

The top-level \texttt{README.md} and \texttt{ARTIFACT\_EVALUATION.md}
are the primary entry points.
\texttt{code/} contains the evaluation scripts
(\texttt{eval\_repro.py} for the classifier, \texttt{gen\_poc.py} for
PoC inference and evaluation), the optional re-training scripts
(\texttt{train\_poc\_generator.py}), and reusability utilities
(\texttt{infer\_batch.py}, \texttt{threshold.py}, \texttt{llm\_final.py},
\texttt{unsafe\_collect.py}).
\texttt{model/} contains the two LoRA adapters and \texttt{reassemble.sh}.
\texttt{scripts/} contains numbered launchers that wrap the underlying
Python commands with the correct environment variables.
\texttt{data/} holds a sample test split; the full datasets are
provided separately via the AE system.

\subsubsection{Security, privacy, and ethical concerns}

Evaluation scripts run read-only inference on the provided datasets.
The PoC evaluation optionally invokes \texttt{cargo build} and
\texttt{cargo +nightly miri run} on generated Rust code inside isolated
\texttt{tempfile} directories; each invocation is short-lived and
self-contained. No network access, persistent file modification, or
elevated privileges are required. The PoCs are designed to trigger
\emph{detected} undefined behavior under Miri, not to exploit any
external system.

\subsubsection{How to access}

The artifact is available at the anonymous repository:
\url{https://anonymous.4open.science/r/coin-ae-E308}.
A Zenodo archive with a permanent DOI will be created upon paper acceptance.
The full \texttt{coin\_test.pkl} (319,652 labeled functions) and
\texttt{poc\_test.jsonl}/\texttt{poc\_train\_v2.jsonl} datasets are submitted
separately through the AE system due to size constraints.

\subsubsection{Hardware dependencies}

A single NVIDIA GPU with $\geq$24\,GB VRAM suffices for E1, E3, and
single-GPU E2. The full E2 run completes in $\approx$10\,hours on one
A100-80\,GB, or $\approx$2.5\,hours on four such GPUs sharded.
Optional PoC re-training requires $\geq$48\,GB VRAM.

\subsubsection{Software dependencies}

Pinned versions (installed automatically by \texttt{scripts/0\_setup\_env.sh}):
\texttt{Python 3.11}, \texttt{unsloth==2025.2.15},
\texttt{transformers==4.49.0}, \texttt{peft==0.14.0}, \texttt{trl==0.15.2},
\texttt{tokenizers==0.21.0}, \texttt{scikit-learn}, \texttt{numpy},
\texttt{matplotlib}, \texttt{tqdm}; CUDA~12.x.
Newer unsloth versions default to a shorter \texttt{max\_seq\_length} when
the argument is omitted, silently truncating Rust code snippets and dropping
AUPRC from $\approx$0.76 to $\approx$0.32. Do not upgrade.
For E3 compile/Miri metrics: Rust stable + nightly toolchain
(\texttt{rustup component add miri --toolchain nightly}).

\subsubsection{Benchmarks}

\textbf{Classifier:} \texttt{coin\_test.pkl} --- 319,652 labeled Rust
functions (97.4\% safe, 2.6\% unsafe) extracted from the top-10K
crates on crates.io using a custom \texttt{rustc 1.83.0-dev} patch
(included as \texttt{custom\_rustc\_patch/rustc.patch}). The test
split is stratified and disjoint from training.
\textbf{PoC generator:} \texttt{poc\_train\_v2.jsonl} --- $\approx$20
manually constructed verified PoC examples per MUF category;
\texttt{poc\_test.jsonl} --- held-out PoC examples for E3.

\subsection{Set Up}

\subsubsection{Installation}

\begin{compactenum}
\item Reassemble split model files:
      \texttt{bash model/reassemble.sh}
\item Create the conda environment and install pinned dependencies:
      \texttt{bash scripts/0\_setup\_env.sh}
\item Place \texttt{coin\_test.pkl} and \texttt{poc\_test.jsonl} at
      local paths.
\end{compactenum}

\subsubsection{Basic test}

Verify model load and CUDA access in $\approx$2 minutes with a
500-sample run:
\begin{verbatim}
CUDA_VISIBLE_DEVICES=0 \
./env/bin/python code/eval_repro.py \
  --checkpoint model/llama3.2 \
  --data /path/to/coin_test.pkl \
  --n 500 --output /tmp/smoke.json
\end{verbatim}
Expected: a non-zero AUPRC printed and \texttt{/tmp/smoke.json} created.
If \texttt{ImportError: bitsandbytes} appears, the launcher scripts
set \texttt{LD\_LIBRARY\_PATH} correctly --- prefer those.

\subsection{Evaluation Workflow}

\subsubsection{Major claims}

\begin{compactitem}
\item[(C1):]
  \textsc{Coin}'s fine-tuned Llama~3.2-3B classifier achieves
  AUPRC\,=\,0.82 on the MUF detection task, and 63.7\% precision
  with 80.4\% recall after PAC-based threshold calibration ---
  substantially outperforming the vanilla Llama~3.2-3B (AUPRC\,=\,0.021),
  QWen3-4B (0.030), GPT-4o, and Claude-3.7.
  Proven by experiments (E1) and (E2); results reported in
  Figure~2 and Table~2 of the paper.

\item[(C2):]
  \textsc{Coin}'s fine-tuned PoC generator produces compilable,
  root-cause-demonstrating PoC code for 19 out of 22 in-the-wild
  MUF bugs found in the crates.io evaluation; the generated PoCs
  were submitted to developers and contributed to 14 confirmed bugs
  and 10 CVE assignments.
  Proven by experiment (E3); results reported in Table~4.

\item[(C3):]
  \textsc{Coin} substantially outperforms vanilla open-source LLMs
  (Llama~3.2 3B, Llama~3.1 8B, Qwen3 4B; AUPRC\,$\approx$\,0.02--0.03)
  and closed-source GPT-4o / Claude-3.7 even with few-shot context
  and Best-of-K querying (precision on the unsafe class never exceeds
  $\approx$7\%).
  Proven by experiment (E4); results reported in Figure~2 and Tables~2--3.
\end{compactitem}

\subsubsection{Experiments}

\begin{compactitem}
\item[(E1):]
  \textbf{Classifier smoke test}
  [5 person-minutes + 20 compute-minutes + 10\,GB disk]:
  Verify the fine-tuned Llama~3.2-3B classifier achieves
  AUPRC\,$\approx$\,0.75 on a stratified 8,000-example sample,
  supporting (C1).

  \begin{asparadesc}
  \item[Preparation:] Complete Setup steps; confirm
        \texttt{unsloth.\_\_version\_\_ == "2025.2.15"}.
  \item[Execution:]
\begin{verbatim}
bash scripts/1_smoke_test.sh \
  /path/to/coin_test.pkl
\end{verbatim}
  \item[Results:] The script prints AUPRC, precision, recall, F1 and
        saves them to \texttt{/tmp/eval\_smoke.json}.
        Expected: AUPRC\,$\approx$\,0.7529,
        recall(unsafe)\,$\approx$\,0.805.
        If AUPRC\,$\approx$\,0.30--0.35, the unsloth version is wrong.
  \end{asparadesc}

\item[(E2):]
  \textbf{Full classifier evaluation}
  [10 person-minutes + 2.5 compute-hours (4-GPU) or 10 compute-hours
  (1-GPU) + 10\,GB disk]:
  Reproduce the AUPRC on the complete 319,652-sample test set,
  supporting (C1).

  \begin{asparadesc}
  \item[Preparation:] Same as E1. The 4-GPU option requires four
        A100-class GPUs (one shard each).
  \item[Execution:]
\begin{verbatim}
# 4-GPU (~2.5 h)
bash scripts/2_full_eval.sh \
  /path/to/coin_test.pkl
# Single-GPU (~10 h)
NUM_SHARDS=1 \
  bash scripts/2_full_eval.sh \
  /path/to/coin_test.pkl
\end{verbatim}
  \item[Results:] Aggregate AUPRC and precision-at-recall-0.8 are
        printed and saved to \texttt{/tmp/eval\_full\_merged.json}.
        Expected: AUPRC\,$\approx$\,0.764 on the fixed stratified split.
        The paper's AUPRC\,=\,0.82 is measured on the natural class
        distribution of the full crate corpus; the $\approx$0.06 gap
        reflects this ratio difference (see ARTIFACT\_EVALUATION.md).
  \end{asparadesc}

\item[(E3):]
  \textbf{PoC generator evaluation}
  [10 person-minutes + 1 compute-hour + 5\,GB disk]:
  Evaluate the fine-tuned PoC generator on held-out MUF examples,
  measuring safe-caller compliance, compilation rate, and
  Miri-detected UB rate, supporting (C2).

  \begin{asparadesc}
  \item[Preparation:] Place \texttt{poc\_test.jsonl} at a local path.
        For \texttt{compiles}/\texttt{ub\_detected}, install Rust and
        nightly Miri (\texttt{rustup component add miri --toolchain nightly}).
        Miri is optional; without it, only \texttt{safe\_caller} is
        meaningful.
  \item[Execution:]
\begin{verbatim}
bash scripts/3_poc_eval.sh \
  /path/to/poc_test.jsonl
\end{verbatim}
        To reproduce fine-tuning from scratch ($\approx$3 h on one A100):
\begin{verbatim}
TRAIN=1 \
TRAIN_DATA=/path/to/poc_train_v2.jsonl \
bash scripts/3_poc_eval.sh \
  /path/to/poc_test.jsonl
\end{verbatim}
  \item[Results:] The script reports per-example and aggregate
        \texttt{safe\_caller} (no \texttt{unsafe} in \texttt{fn main()}),
        \texttt{compiles} (\texttt{cargo build} passes), and
        \texttt{ub\_detected} (\texttt{miri} flags UB), saved to
        \texttt{/tmp/poc\_eval.json}. The fine-tuned model generates
        valid PoC skeletons for the majority of test examples,
        consistent with the 19/22 success rate in Table~4.
  \end{asparadesc}

\item[(E4):]
  \textbf{Baseline comparison}
  [15 person-minutes + 2 compute-hours + a few US\$ in API credits + 30\,GB
  disk]: Reproduce the comparison against vanilla open-source LLMs
  (paper Fig.~2) and closed-source GPT-4o / Claude-3.7 (paper
  Tables~2--3), supporting (C3).

  \begin{asparadesc}
  \item[Preparation:] Export the API keys you have available
        (\texttt{OPENAI\_API\_KEY}, \texttt{ANTHROPIC\_API\_KEY}); the
        launcher skips a provider whose key is missing. Llama models
        on HuggingFace are gated --- set \texttt{HF\_TOKEN} or run
        \texttt{huggingface-cli login} once. The provided
        \texttt{data/shots.jsonl} contains five labeled examples for
        zero-friction few-shot runs; larger shot files follow the
        format documented at the top of \texttt{eval\_api\_baseline.py}.
  \item[Execution:]
\begin{verbatim}
bash scripts/4_baseline_eval.sh \
  /path/to/coin_test.pkl
\end{verbatim}
        Or one configuration directly:
\begin{verbatim}
# Open-source: Llama 3.2 3B vanilla
./env/bin/python \
  code/eval_open_baseline.py \
  --model meta-llama/Llama-3.2-3B-Instruct \
  --data /path/to/coin_test.pkl \
  --n 8000 --output /tmp/open_llama32.json
# API: GPT-4o, N=3 few-shot, Best@1
./env/bin/python \
  code/eval_api_baseline.py \
  --provider openai --model gpt-4o \
  --data /path/to/coin_test.pkl --n 200 \
  --shots_file data/shots.jsonl --shots_n 3 \
  --best_of 1 --output /tmp/gpt4o_n3.json
\end{verbatim}
        The launcher iterates N\,$\in\,\{0,1,3,5\}$ (Table~2) and N=1
        with K\,$\in\,\{3,5\}$ (Table~3) for each provider.
  \item[Results:] Per-config JSON files in \texttt{/tmp/coin\_baselines/}.
        Expected: vanilla open-source AUPRC $\approx$ 0.02--0.03
        (Fig.~2); GPT-4o precision/recall on the unsafe class
        $\approx$\,4--5\,\% / 21\,\%; Claude-3.7 precision/recall
        $\approx$\,4--7\,\% / 7--12\,\%, matching the order of
        magnitude in Tables~2--3. Coin's reproduced
        precision/recall = 63.7\,\% / 80.4\,\% (E2) is one to two
        orders of magnitude better.
  \end{asparadesc}

\end{compactitem}

\subsection{Notes on Reusability}

\texttt{code/infer\_batch.py} runs the classifier on any Rust crate
directory and outputs a per-function CSV of MUF probabilities, enabling
application to new codebases. \texttt{code/threshold.py} provides
PAC-based threshold calibration for specific
precision/recall operating points. \texttt{code/gen\_poc.py --infer}
accepts arbitrary MUF function text and category labels, allowing the
PoC generator to be applied to MUFs found by other tools. Both adapters
load via standard PEFT/HuggingFace APIs independently of the included
scripts, facilitating integration into custom workflows.
